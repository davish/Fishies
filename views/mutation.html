Mutation is another process that's essential for our genetic algorithm. In the mutation process, the chromosomes resulting from the crossover process are mutated by a small amount. The idea behind this process is that small variations may give some chromosomes a higher probability of success, and will persist to later generations.

Many genetic algorithms have a "mutation rate," which is the probability for any particular value in the chromosome to be randomly modified. Because all of our values are on a continuous scale, it makes more sense to adjust all values by a relatively small amount. We add a random number to each of the values in each chromosome. The random number is normally distributed, meaning that it is more likely to be a small change than a large one.

The standard deviation of this distribution is proportional to the current value. This means that if you set the "Mutation" setting to 5%, then there is a 68% chance that the mutated value will be within 5% of the original one. If the mutation setting is too low, then the population will evolve very slowly, and if it is too high, then it may be unstable.

If the "Dynamic Mutation Rate" option is enabled, then the mutation rate will slowly decrease over the course of the simulation. This is similar to the process of simulated annealing. The idea is that the population will evolve quickly originally, but will stabilize later.
