The Genetic Fishbowl uses what is called a genetic algorithm, so called because it emulates biological processes. Each fish has a "chromosome." In biology, this chromosome would be a series of nitrogen bases, but in our case, it's a series of values that affect a fish's appearance and behavior. As some fish succeed and others die, the whole population slowly evolves to adapt to the environment. In our simulation, the system is interactive, so the user may "breed" fish if they so desire. Here's an overview of the algorithm:

Initial population

At the beginning of the simulation, a population of fish is created, each with a random chromosome. To learn more about the chromosomes, visit the Fish page. The population size can be customized in the settings panel. During this phase (and for each generation), the bowl is also populated with food, proportional to the population size. Each food pellet has 10 energy points.

User Interaction

The user can interact with the fish in a variety of ways. The user can kill unwanted fish by left-clicking, and can place food in the bowl with a right-click. Note that if "Generation Mode" is set to "Iterative," then each generation will end when either all the food runs out or half of the population dies.

Fitness Score

Each fish is given a "fitness score" to be used in the selection process. If "Generation Mode" is set to "Iterative," the fitness score is based on how many energy points a fish has. Energy points are gained when the fish eat food, and are lost when fish move. If "Generation Mode" is set to "Continuous," then the fitness score is based on how long a particular fish has been alive.
