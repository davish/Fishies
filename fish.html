Each of the fish in our simulation has a "chromosome." In biology, this chromosome would be a series of nitrogen bases. While some other genetic algorithms encode characteristics in binary and represent the chromosome as a series of ones and zeros, we decided to simplify our model. Each fish's chromosome is an array of __ numerical values that control various aspects of its appearance and behavior. A typical chromosome may look like this:

The next six values affect a fish's color. The first three represent the RGB values of the fish body, and the next three represent the RGB values of the fish's secondary color. The secondary color is what's used for the outline around the fish and for the fish's eye. Each of these values is on a scale from 0 to 255. A fish with the first six values of [250, 2, 8, 255, 0, 0, 0, 0, 255] would have a bright red body and a bright blue outline.

Aside from these dimensional values, the chromosome also contains several values that affect a fish's behavior. The tenth value in the chromosome is the "exploration" value. This means that if the fish sees nothing of interest nearby, this is the probability each frame that the fish will pick a random direction to travel. If this value is too low, the fish will only travel in straight lines; if it is too high, the fish may practically spin in circles.

The final three values are the fish's priorities for various goals. The first goal is to eat small fish, the second goal is to avoid being eaten by big fish, and the third goal is to eat food. Each frame, each fish makes a decision about which direction to travel.
